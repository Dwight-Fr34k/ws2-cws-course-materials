Before we start

Prerequisites (1)

  • HTML5
    • base document structure
    • correct use of elements and attributes
    • HTML markup for forms
    • correct sectioning & header numbering (outliner tool)
    • new HTML5 features
  • CSS3
    • full spectrum of selectors and techniques
    • em/rem based layout
    • layout techniques
    • resonsive layout with media queries
    • new CSS3 features
CSS checklist
  • best practices
    • consistent CSS files structure with comments
    • class attributes prefered over id attributes
    • short selectors, i.e. #mainmenu ul li a
  • selectors
    • basic selectors (tag, .class, #id)
    • pseudo class selectors (:first-child, :hover...)
    • pseudo element selectors (::first-letter, ::after...)
    • grouped selectors (A, B)
    • contextual selectors (A B)
    • CSS 2.1 selectors (*, A > B, A + B...)
    • attribute selectors (A[a], A[a=x], A[a~=x]...)
  • properties & fonts
    • em/rem
    • priority and inheritance
    • Google fonts, webfonts
  • layout
    • box model, box-sizing
    • absolute/relative/fixed/static
    • float/clear/clearfix
    • table → not ready yet
    • flexbox → not ready yet
  • CSS3
    • new selectors (nth-of-type, nth-child...)
    • background sizes, color definitions, gradients
    • transforms, transitions, animations
    • box (box-shadow, border-radius, box-sizing...)
  • responsive & media queries
HTML checklist
  • code conventions
    • lowercase tags, i.e. <TABLE> → <table>
    • double quotes around attributes
    • proper nesting
    • close all non empty tags, i.e. <li>...</li>
    • allowed: open empty tags, i.e. <br> or <br />
    • allowed: attribute minimalization, i.e.
      <input selected="selected" ...> or
      <input selected ...>
  • base document
    • <!doctype html>
    • <title> provided
    • <meta charset="..."> provided
  • classic HTML elements <p>, <img> etc...
  • forms
    • general HTML structure with <dl>'s or <p>'s, checkboxes and radiobuttons nested inside <label>
    • for and name attributes
  • sectioning
    • correct use of <div>, <section>, <nav>...
    • correct numbering of <h1>, <h2>... (see outline)
  • HTML5
    • new input types date, color, email, range...
    • HTML5 elements (<figure>, <audio>, <video>)
    • attributes (contenteditable, data-*)

Prerequisites (2)

  • MySQL
    • basic CRUD actions (INSERT, SELECT, UPDATE, DELETE)
    • subqueries & joins (LEFT, RIGHT, INNER)
    • searches (LIKE, MATCH...AGAINST, FULLTEXT)
    • options (ORDER, LIMIT)
    • basic administration with PhpMyAdmin or alike

Prerequisites (3)

  • PHP
    • basic syntax
    • client-server
    • classes
    • forms (GET/POST, formchecking, processing, file uploads)
    • files and folders (looping, read/write, file info)
    • databases with PDO (connecting, CRUD, prepared statements, fetching results)
    • templates with Twig (displaying, filters, loops, options, master/parent...)
    • persistence (cookies, sessions, login forms)
    • summaries in serverside webscripten 1 course materials (Git)

Prerequisites (4)

  • Javascript
    • basic syntax (incl. anonymous functions & objects, object & array literals, scope, closures)
    • initializing scripts (window load event, encapsulation)
    • DOM tree (finding and manipulating elements & properties/CSS)
    • event handling (attaching/removing, bubbling, target, preventing defaults)
    • jQuery basics
    • AJAX (creating/consuming PHP API, JSON)
    • integrating and consuming external libraries and API's
    • clientside webscripten 1 course materials (ikdoeict.be)
    • w3schools DOM reference
    • jQuery and jQueryui documentation

Prerequisites (5)

  • Programming
    • iterations (for) & selections (if, if - else)

    • functions
    • basic OO concepts:
      • classes
      • constructors
      • instances
      • properties
      • inheritance
      • static v.s. non-static (and when to use which)

Prerequisites (6)

  • General
    • proper coding style
      • layout
      • consistency
      • efficiency
    • proper documentation
      • consistent commenting, Javadoc-inspired
      • readme.md

      • Git

Webdeveloper checklist
  • Shift-Ctrl-I or F12
  • code inspection
    • DOM window
    • styles windows (rules, properties, box model)
    • live edit, copy, delete...
  • console
    • console.log
    • Javascript errors
    • HTTP errors (code >= 400)
    • statement execution
  • network
    • requests & responses inspection
  • resources
    • cookies, local storage, Web SQL...
  • debugger
    • adding breakpoints (sources tab)
    • variable inspection
    • step into/out/over

Software

Course Philosophy

  • This course isn't
    • a tutorial on Bootstrap or Angular
    • a training of specific techniques
    • a hands-on manual on development tools
  • Instead, it will try to give
    • a generic helicopter view of current and future technologies
    • the right pointers to make your own choices
    • the opportunity to practice
    • the freedom to develop your own style
  • leaves more responsibility to you, might not be for everyone

Help!

  • Debugging
    • dump variables with console.log()
    • debug using breakpoints and variables inspection
    • Google your error message
    • remove code till the problem disappears; inspect the last chunk removed
    • inspect call stack (less useful with events and callbacks-driven Javascript)
  • Get help
    • if you're stuck without making any progress whatsoever, timing rules:
      • <10 min: keep trying
      • >10 min, co-student or the instructor around: you must ask for help
      • >20 min, colleague around: you must ask for help
      • >60 min, no-one around: you must ask for online help
    • online help: post a question on ICT-Blue FB group; include a linked name of your instructor for swift response
    • e-mail is for personal matters only (allow 3 working days for a response)
